Note: 6-digit entries are either ROMPTRs or FLASHPTRs.
When using LIBEVAL, remember to swap the first three digits
with the last 3 digits; e.g. for 012345, use #345012 LIBEVAL.
The FLASHEVAL values work as-is.
6-digit entries in the HP48 column are for the 48G/GX only.

ROMPTR names are preceded by ~ (e.g. ~FOO)
FLASHPTR names are preceded by ^ (e.g. ^FOO)
User RPL commands are preceded by x (e.g. xFOO).
Thus, ^xFOO would be a FLASHPTR User RPL command.
Omit these leading characters when searching alphabetically;
all of the above examples are alphabetized as "FOO".

Keep in mind that command names were limited to 12 characters in HP48
System RPL and Assembly Language, but HP49 names can be longer, so be
sure to truncate names to 12 characters when searching for a 48 name.
For example, the following two objects are really the same:
HP48: ~grobAlertIc
HP49: ~grobAlertIcon

Note: FLASHPTR's 102007 through 11C007 appear in this table, but are
valid only in ROM versions 1.11 and higher.
They generate the "Undefined FPTR Name" error (ERRN: #11h) if used in
ROM versions 1.10 or below.

Although this is advertised as a list of "supported" entry points,
there are many entries that are supported in the 49 that were never
supported in the 48 *but* remained at the same ROM address in all
versions of the 48 (A through R).  These entries can be used with
confidence in the 48 and 49, and are marked "U.B.S." (Unsupported
But Stable in the 48; supported in the 49).

"ML" in the comment area means that it's a Machine Language entry
point, not the address of an RPL object.  Do not use SYSEVAL on them!
